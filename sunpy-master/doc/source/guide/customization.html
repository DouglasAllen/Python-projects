Sunpy uses :file:`sunpyrc` configuration files to customize certain properties. You can control a number of key features of SunPy such as where your data will download to. SunPy looks for :file:`sunpyrc` in two locations, in the following order:

  1. :file:`.sunpy/sunpyrc`, for the user's default customizations.

    System Message: ERROR/3 (customization.rst, line 17); backlink

  2. :file:`{INSTALL}/sunpy/data/sunpyrc`, where :file:`{INSTALL}` is something like :file:`/usr/lib/python2.5/site-packages` on Linux, and maybe :file:`C:\\Python25\\Lib\\site-packages` on Windows. Every time you install SunPy, this file will be overwritten, so if you want your customizations to be saved, please move this file to your :file:`.sunpyrc` directory.

To display where the currently active :file:`sunpyrc` file was loaded from, one can do the following:

System Message: ERROR/3 (customization.rst, line 25); backlink

Unknown interpreted text role "file".
>>> import sunpy
>>> sunpy.print_config()

The default sunpyrc file lives in sunpy/data/. We recommend that you make a copy of this file and put it where Sunpy will find it. See below for the example config file.

Dynamic settings

You can also dynamically change the default settings in a python script or interactively from the python shell. All of the settings are stored in a Python ConfigParser instance called :data:`sunpy.config`, which is global to the sunpy package. Settings can be modified directly, for example:

System Message: ERROR/3 (customization.rst, line 39); backlink

Unknown interpreted text role "data".
import sunpy
sunpy.config.set('downloads', 'download_dir', '/home/user/Downloads')
